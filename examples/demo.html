Parent-Child UI Element Management with Cascading Styles

📚 Parent-Child Concept

QElements allows you to define parent styles in one central file, and have all child components inherit those styles automatically.

// Parent Definition (main-parents.tsx) const mainContainer = new QElement('main_container', { padding: 20, backgroundColor: '#f0f0f0', border: '2px solid #007acc' }); // Child Usage (app.tsx) <QElementComponent elementId="main_container"> <h1>This inherits parent styles!</h1> </QElementComponent>
💡 Key Benefit: Change the parent style once, and ALL children using that parent get updated automatically!

✨ Architecture Benefits

🎯 Centralized Control

All parent styles defined in one file (main-parents.tsx)

🔄 Automatic Inheritance

Children automatically inherit parent styles

🎨 Individual Overrides

Each child can override specific styles when needed

🔍 No File Searching

Change parent once, affects all children globally

📁 File Structure

src/ ├── styles/ │ └── main-parents.tsx # 🎯 ALL PARENT STYLES ├── components/ │ ├── app.tsx # Child using main_container │ ├── mainapp.tsx # Child using main_container │ ├── qelementtooltips.tsx # Child using card_container │ └── dashboard.tsx # Child using dashboard_layout
